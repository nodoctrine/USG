0.1 Bitwise Operators

C provides six bitwise operators that operate on integer values one bit at a time. Unlike arithmetic operators, they work directly on the binary representation and do not care about sign or magnitude — they just flip, combine, or shift individual bits.

  • & (AND) — output bit is 1 only if both input bits are 1. Used to isolate or test bits.
  • | (OR) — output bit is 1 if either input bit is 1. Used to set bits.
  • ^ (XOR) — output bit is 1 if the input bits differ. Used to toggle bits.
  • ~ (NOT) — flips every bit. Used to build inverted masks for clearing bits.
  • << (left shift) — moves all bits toward the high end; vacated bits fill with 0. Multiplies by 2 per shift.
  • >> (right shift) — moves all bits toward the low end. Divides by 2 per shift (unsigned values).

These operators are foundational in embedded C, where every bit of a hardware register often controls a distinct piece of behavior.

Common use: extract a field from a register by ANDing with a mask that is 1 only in the target bit positions.
OR  ( | ) — Set Bits

  a = 1011 1010
  b = 0000 0001  (0x01)
a|b = 1011 1011

Bit 0 is now 1. All other bits are unchanged. OR can only turn bits on — it can never clear a bit that is already 1.
XOR  ( ^ ) — Toggle Bits

  a = 1011 1010
  b = 0000 1111  (0x0F)
a^b = 1011 0101

Every bit in the lower nibble flipped. The upper nibble (masked with 0) was left alone. XOR with the same mask again restores the original value.
Shifts  ( <<   >> ) — Move Bits

   a = 0000 0001  (1)
a<<3 = 0000 1000  (8)  ✓ bit 3 set
a<<5 = 0010 0000  (32) ✓ bit 5 set

(1 << N) is the standard idiom for a mask with only bit N set. Every bit manipulation pattern in the next section uses this.

0.2 Bit Masking Patterns

A bit mask is an integer value used as a template to manipulate specific bits in another value. The four universal patterns — set, clear, toggle, check — are the building blocks of all register-level programming.

Every pattern uses (1 << N) as the mask for bit N:

  • Set bit N: x |= (1 << N) — forces bit N to 1, leaves all others unchanged
  • Clear bit N: x &= ~(1 << N) — forces bit N to 0, leaves all others unchanged
  • Toggle bit N: x ^= (1 << N) — flips bit N, leaves all others unchanged
  • Check bit N: (x >> N) & 1 — returns 1 if bit N is set, 0 if clear

To operate on multiple bits at once, combine masks with OR: (1 << 3) | (1 << 5) creates a mask with bits 3 and 5 set simultaneously.

The compound assignment forms (|=, &=, ^=) are essential — they read the current value, apply the change, and write back. This read-modify-write sequence is the only safe way to update a register without corrupting the bits you did not intend to touch.

These four patterns cover every single-bit operation. Multi-bit operations combine masks: x |= (1<<2)|(1<<5) sets bits 2 and 5 simultaneously.

0.3 Hardware Register Control

In embedded and systems programming, hardware registers are fixed memory addresses mapped directly to control circuits inside the microcontroller or peripheral. Writing a specific bit to one of these addresses instantly changes a physical pin state, timer setting, interrupt enable, or clock source.

Two rules govern all register access in C:

  • Always use volatile. Declare register pointers as volatile so the compiler never caches a read or eliminates a write as "redundant." Hardware can change a register's value independently of your code — the compiler must not assume the value stays constant between accesses.
  • Always read-modify-write. Never write a literal value directly to a control register (REG = 0x08) — this zeros every bit you did not mention, potentially disabling unrelated hardware. Always use |= to set and &= ~(mask) to clear.

Most vendor SDKs define register names and bit positions as macros, so your code reads like GPIOA->ODR |= GPIO_PIN_5 rather than *(uint32_t*)0x40020C14 |= (1<<5) — but the underlying operation is identical.

Figure Figure 0.3 — Example 8-bit GPIO Control Register
PORT_CTRL  (address: 0x40021000)
Bit 7 Bit 6 Bit 5 Bit 4 Bit 3 Bit 2 Bit 1 Bit 0
RESET IRQ_EN reserved reserved DIR_1 DIR_0 EN_1 EN_0
// Enable channel 0, set direction 0 to output, enable IRQ
PORT_CTRL |= (1<<0) | (1<<2) | (1<<6);

// Disable channel 0 without touching anything else
PORT_CTRL &= ~(1<<0);
Reserved bits must never be written — always use read-modify-write so the compiler touches only the bits your code names.
